概览

The Objective-C runtime is a runtime library that provides support for the dynamic properties of the Objective-C language, and as such is linked to by all Objective-C apps. Objective-C runtime library support functions are implemented in the shared library found at /usr/lib/libobjc.A.dylib.

Apple所描述的runtime是一个为Objective-C提供运行时属性的库。

运行时(runtime)

运行时rumtime就是程序运行的时候。
运行时库就是程序运行的时候所需要依赖的库。
运行的时候指的是指令加载到内存并由CPU执行的时候。

消息转发(message_forward)

消息转发第一步会调用

允许此时给Class动态添加实现，如果有实现了，调用返回；如果仍没有实现，就继续调用下面这个方法

允许返回一这个能响应该消息的对象，如果获取到了，向新对象转发消息；返回nil的话，则继续的方法获取一个方法签名

如果获取不到，就直接调用doesNotRecognizeSeletor:抛出异常；获取到了方法签名就继续调用

上面获取到的方法签名包装成Invocation传入，如何处理就在这里面了。

Method Swizzling

Objective-C中的AOP，就是利用了runtime的特质。

Safe

现在大部分利用runtime做crash safe都是直接swizzling原方法。这么做的有一定的几率会出现莫名的bug，原因是一些系统库并不一定都是正常的引用NSArray、NSDictionary、NSString等

1.NSNull

Java会将空字符串在json中用null来表示，所以有的时候会在模型类中出现某些NSString或者其他类变成NSNull的情况，这时候如果没有判断是否为NSNull就调用NSString等其他类的方法，就会抛出doesNotRecognizeSeletor:异常了，出现App crash。

根据消息转发机制，可以利用forwardingTargetForSelector:来做安全。
